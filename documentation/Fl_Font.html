typedef Fl_Font

The font used by Fl_Widget::label_font() and other parts of fltk is a typedef for a ponter to the structure Fl_Font_.

The following Fl_Font values are predefined:

  • FL_HELVETICA
  • FL_HELVETICA_BOLD
  • FL_HELVETICA_ITALIC
  • FL_HELVETICA_BOLD_ITALIC
  • FL_COURIER
  • FL_COURIER_BOLD
  • FL_COURIER_ITALIC
  • FL_COURIER_BOLD_ITALIC
  • FL_TIMES
  • FL_TIMES_BOLD
  • FL_TIMES_ITALIC
  • FL_TIMES_BOLD_ITALIC
  • FL_SYMBOL
  • FL_SCREEN
  • FL_SCREEN_BOLD
  • FL_ZAPF_DINGBATS (different between X and Windows!)

Methods

Fl_Font_* Fl_Font_::bold;
Fl_Font_* Fl_Font_::italic;

Pointers to the bold and italic versions of this font.

const char* Fl_Font_::system_name()

Returns a string that identifies the font in a system-specific manner. This string is guaranteed to be different for each font but it is not portable. Under X this value is passed to XListFonts to get all the sizes of this font.

const char* Fl_Font_::name(int* attributes = 0)

Returns a "nice" name for the font. This is useful if you are presenting a choice to the user. There is no guarantee that each face has a different name. The return value points to a static buffer that is overwritten each call.

The integer pointed to by attributes is set to zero, FL_BOLD or FL_ITALIC or FL_BOLD | FL_ITALIC. If this pointer is zero then the attributes are indicated by adding "Bold" or "Italic" to the name.

int Fl_Font_::sizes(int *&sizep) const;

Return an array of sizes in sizep. The return value is the length of this array. The sizes are sorted from smallest to largest and indicate what sizes can be given to fl_font() that will be matched exactly (fl_font() will pick the closest size for other sizes). A zero in the first location of the array indicates a scalable font, where any size works, although the array may list sizes that work "better" than others. Warning: the returned array points at a static buffer that is overwritten each call. Under X this will open the display.

Fl_Font fl_font(const char* name, int attributes = 0);

Find a font with the given "nice" name. You can get bold and italic by either putting "Bold" or "Italic" in the name, or by passing them as the attributes. Case is ignored and fltk will accept some variations in the font name. Not yet implemented...

int fl_list_fonts(Fl_Font*& arrayp, bool everything = false);

FLTK will open the display and find every font on the server. It creates an array of these and returns that array in arrayp, and the length as the return value. It will attempt to put "families" of faces together, so that the normal one is first, followed by bold, italic, and bold italic.

The optional argument everything will cause fltk to list every single font on an X server. Normally it ignores fonts whose X names don't start with '-'. This can be used to get the bitmap fonts that are useful for terminal emulators. This flag does nothing on Windows.

Subsequent calls always return the same array, and the value for everything is ignored.

extern const char* fl_encoding;

You can change this string to change the "preferred" encoding. If a given font is available in multiple encodings this one is returned. Unlike other drawing toolkits this does not prevent the font from being found if the given encoding is not available, it will just return some random encoding (in my opinion this behavior is what the user wants).
